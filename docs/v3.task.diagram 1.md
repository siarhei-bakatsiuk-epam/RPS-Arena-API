# Диаграммы: RPS Arena API

---

## 1. Общая архитектура системы

```mermaid
graph TB
    Client["Client (HTTP)"]

    subgraph MatchService
        MS_API["API Layer"]
        MS_App["Application\n(CQRS / MediatR)"]
        MS_Domain["Domain"]
        MS_Infra["Infrastructure\n(EF Core)"]
    end

    subgraph LeaderboardService
        LS_API["API Layer"]
        LS_App["Application\n(CQRS / MediatR)"]
        LS_Domain["Domain"]
        LS_Infra["Infrastructure\n(EF Core)"]
    end

    Broker[("Message Broker\nRabbitMQ / Kafka")]
    DB1[("PostgreSQL\nMatch DB")]
    DB2[("PostgreSQL\nLeaderboard DB")]

    Client -->|REST| MS_API
    Client -->|REST| LS_API

    MS_API --> MS_App --> MS_Domain
    MS_Infra -->|implements| MS_Domain
    MS_Infra --> DB1
    MS_Infra -->|publish MatchRecorded| Broker

    Broker -->|consume| LS_Infra
    LS_API --> LS_App --> LS_Domain
    LS_Infra -->|implements| LS_Domain
    LS_Infra --> DB2
```

## 2. Слоистая архитектура (зависимости)

```mermaid
graph LR
    Infra["Infrastructure"]
    Domain["Domain"]
    App["Application"]
    API["API"]

    Infra -->|depends on| Domain
    App -->|depends on| Domain
    API -->|depends on| App
```

> Domain не зависит ни от чего. Infrastructure и Application зависят от Domain. API зависит от Application.

## 3. Запись матча (sequence)

```mermaid
sequenceDiagram
    autonumber
    participant Client
    participant MatchService
    participant MatchDB as PostgreSQL (Match)
    participant Broker as Message Broker

    Client->>MatchService: POST /matches
    MatchService->>MatchService: FluentValidation
    MatchService->>MatchDB: Idempotency check + INSERT match
    MatchDB-->>MatchService: OK
    MatchService->>Broker: Publish MatchRecorded
    MatchService-->>Client: 201 Created
```

---

## 4. Асинхронный пересчёт лидерборда (sequence)

```mermaid
sequenceDiagram
    autonumber
    participant Broker as Message Broker
    participant LeaderboardService
    participant LeaderboardDB as PostgreSQL (Leaderboard)
    participant Client

    Broker->>LeaderboardService: MatchRecorded event
    LeaderboardService->>LeaderboardDB: BEGIN TRANSACTION
    LeaderboardService->>LeaderboardDB: SELECT player stats (lock)
    LeaderboardService->>LeaderboardService: Recalculate wins/draws/losses\nmatchPoints / totalScore
    LeaderboardService->>LeaderboardDB: UPDATE player stats
    LeaderboardDB-->>LeaderboardService: COMMIT

    Client->>LeaderboardService: GET /leaderboard?sortBy=matchPoints&top=10
    LeaderboardService-->>Client: Top players list
```
